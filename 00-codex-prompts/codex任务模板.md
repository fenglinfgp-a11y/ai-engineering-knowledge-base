# Codex 任务模板

来源类型：实战总结

## 问题场景

需要让 Codex 处理一个明确工程任务，但又要限制它不要扩大范围、不要误改无关文件。

## 判断方法

- 任务是否有明确目标。
- 是否说明允许修改的目录。
- 是否说明禁止修改的内容。
- 是否要求验证方式。

## 操作步骤

可复制模板：

```text
你现在只允许在当前仓库内工作，不要修改其他项目。

目标：
<写清楚要完成什么>

限制：
1. 不要改无关文件。
2. 不要保存密钥、Token、客户资料。
3. 如果发现现有改动，不要回滚，先说明。
4. 所有命令都要说明用途。

请完成：
1. 先查看项目结构。
2. 给出最小实现方案。
3. 修改必要文件。
4. 运行验证命令。
5. 最后总结修改文件和验证结果。
```

## 验证方法

- Codex 输出中能看到它先检查了目录。
- 修改文件数量与任务范围一致。
- 最后有验证命令和结果。

## 注意事项

- 不要一次给 Codex 过多无关目标。
- 涉及生产环境前，必须要求只读检查，不要直接执行变更。

## 适用范围

Codex 指令模板、任务边界控制和 AI 辅助维护流程。

## 相关命令

`pwd`、`git status --short`、`git diff --stat`。
