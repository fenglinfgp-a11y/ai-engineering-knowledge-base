# 只分析不修改模板

来源类型：实战总结

## 问题场景

只想让 Codex 看项目、查问题、给建议，不希望它直接改代码或改配置。

## 判断方法

- 任务描述里是否包含“只分析”“不要修改”。
- 是否需要列出风险、影响范围、建议步骤。

## 操作步骤

可复制模板：

```text
只分析，不要修改任何文件，不要执行写入命令。

请检查：
1. 问题可能出在哪里。
2. 涉及哪些文件或配置。
3. 建议的最小修改方案。
4. 修改前应该备份或确认什么。

输出要求：
- 按严重程度列出发现。
- 给出文件路径和原因。
- 不要直接改文件。
```

## 验证方法

- `git status --short` 没有新增或修改文件。
- Codex 没有执行 `apply_patch`、重定向写入、删除、移动等操作。

命令说明：

```bash
git status --short
```

用途：查看当前仓库是否有文件变更。

## 注意事项

- 如果 Codex 发现高风险问题，也只能说明，不应直接修复。
- 需要修复时，另开一条明确的修改指令。

## 适用范围

Codex 指令模板、任务边界控制和 AI 辅助维护流程。

## 相关命令

`pwd`、`git status --short`、`git diff --stat`。
