# 更新索引模板

来源类型：实战总结

## 问题场景

新增或调整文档后，需要同步更新索引，避免知识库变成“有文件但找不到”。

## 适用范围

适用于新增 cases、commands、checklists、workflows、security、Codex 模板、技术文档后的索引维护。

## 判断方法

- `git status --short` 显示有新增或修改文档。
- 新文档还没有出现在 `99-index/技术索引.md`、`99-index/问题排查索引.md` 或 `99-index/常用命令索引.md`。
- 文档主题能对应到问题、命令或维护流程。

## 操作步骤

可复制给 Codex：

```text
请只在 /Users/dora/Documents/ai-engineering-knowledge-base 中操作。

请检查本次新增/修改的 Markdown 文件，并更新：
1. 99-index/技术索引.md
2. 99-index/常用命令索引.md
3. 99-index/问题排查索引.md

要求：
- 技术索引按问题或主题加入入口。
- 常用命令索引只加入命令类、流程中常查命令、风险命令分级入口。
- 问题排查索引按故障现象加入入口。
- 不要改业务项目，不要写敏感信息。
- 更新后执行 find . -name "*.md" -size 0 和 git status。
```

## 验证方法

- 新文档能从至少一个索引入口找到。
- 排查类文档能从 `问题排查索引.md` 找到。
- 命令类文档能从 `常用命令索引.md` 找到。

## 注意事项

- 索引不要堆砌无关入口，优先按常见问题归类。
- 同一个文档可以出现在多个索引，但描述要清楚用途。

## 相关命令

```bash
git status --short
```

用途：查看本次新增或修改了哪些文档。

```bash
find . -type f | sort
```

用途：查看知识库完整文件结构，避免漏掉新增文档。
