# 源码交付模板

来源类型：实战总结

## 问题场景

需要检查一个项目是否具备可开发、可部署、可交付条件。

## 判断方法

- 是否包含 backend、frontend、database 三件套。
- 是否有启动说明、环境变量示例、依赖版本。
- 是否排除了密钥、缓存、构建产物、真实数据。

## 操作步骤

可复制模板：

```text
请检查这个源码交付包是否完整，不要修改业务代码。

重点检查：
1. backend 是否可安装依赖并启动。
2. frontend 是否可安装依赖并打包。
3. database 是否有结构 SQL 或迁移文件。
4. 是否包含 README、环境变量示例、部署说明。
5. 是否包含不该交付的密钥、日志、缓存、真实数据。

输出：
- 缺失项
- 风险项
- 建议补齐文件
- 可交付结论
```

## 验证方法

- 后端能在测试环境启动。
- 前端能成功构建。
- 数据库能在空库导入结构。
- `.env` 只提供示例，不包含真实密钥。

## 注意事项

- 不要把 `node_modules`、`vendor`、`target`、`dist` 作为源码交付核心内容。
- 真实数据库备份必须脱敏后才可交付。
