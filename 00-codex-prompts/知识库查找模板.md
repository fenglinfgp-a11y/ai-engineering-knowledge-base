# 知识库查找模板

来源类型：实战总结

## 问题场景

需要让 Codex 只在本知识库中查找已有经验，不修改文件、不读取业务项目。

## 适用范围

适用于查找部署、排查、交付、Android/iOS 调试、数据库维护、Codex 指令模板等已有文档。

## 判断方法

- 当前目录必须是 `/Users/dora/Documents/ai-engineering-knowledge-base`。
- 优先查 `99-index/技术索引.md`、`99-index/问题排查索引.md`、`99-index/常用命令索引.md`。
- 如果没有匹配文档，只输出建议补充位置，不创建新文件。

## 操作步骤

可复制给 Codex：

```text
请只在 /Users/dora/Documents/ai-engineering-knowledge-base 中查找资料，不修改任何文件，不读取其他项目。

我要查找的问题是：<填写问题>

请按以下顺序查找：
1. 99-index/技术索引.md
2. 99-index/问题排查索引.md
3. 99-index/常用命令索引.md
4. cases/、commands/、checklists/、workflows/、security/ 中相关文档

输出要求：
- 告诉我应该看哪些文档。
- 摘要说明每个文档能解决什么问题。
- 如果没有现成文档，建议应该新增到哪个目录。
- 不要修改文件，不要提交 git。
```

## 验证方法

- Codex 输出的路径都在知识库目录内。
- 没有产生 `git status --short` 变更。
- 回答能定位到具体文档和后续补充位置。

## 注意事项

- 不要让 Codex 搜索业务项目。
- 不要粘贴真实日志、密钥、客户资料；如需提供日志，先脱敏。

## 相关命令

```bash
pwd
```

用途：确认当前目录是否为知识库。

```bash
git status --short
```

用途：确认查找任务没有产生文件改动。
