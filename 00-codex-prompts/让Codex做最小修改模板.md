# 让 Codex 做最小修改模板

来源类型：实战总结 / 我的项目经验

## 问题场景

希望 Codex 只改知识库中某个小问题，例如补一句说明、修正命令用途、更新索引，不做无关重构。

## 适用范围

- 适用于知识库小范围维护。
- 适用于修复错别字、补充风险提示、补充参考来源。
- 不适用于业务项目代码修改。

## 判断方法

- 明确文件路径和修改目标。
- 明确禁止改动范围外文件。
- 修改后必须检查 diff 和状态。

## 操作步骤

可直接复制以下提示词：

```text
请只在 /Users/dora/Documents/ai-engineering-knowledge-base 内操作。
不要读取或修改业务项目。
只允许修改这些文件：
- 【文件路径 1】
- 【文件路径 2】

修改目标：
【写清楚要改什么】

限制：
1. 不做无关重构。
2. 不新增空文档。
3. 不写敏感信息或违法绕过内容。
4. 命令必须解释用途，危险命令标记风险。
5. 修改后执行 git diff --stat、find . -name "*.md" -size 0、git status。
```

用途：把 Codex 的修改范围压到最小。

## 验证方法

- `git diff --stat` 只包含指定文件或必要索引。
- 空 Markdown 检查无输出。
- `git status` 中没有业务项目文件。

## 注意事项

- 如果发现需要扩大范围，先停下来说明原因。
- 不确定的内容不要补细节，先标注待确认。

## 相关命令

`git diff --stat`、`find . -name "*.md" -size 0`、`git status`。
