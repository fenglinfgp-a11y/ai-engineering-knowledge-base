# 让 Codex 学习资料模板

来源类型：实战总结 / 我的项目经验

## 问题场景

希望 Codex 根据公开官方文档、开源项目和公开社区资料补充知识库，但不写入敏感信息或违法绕过内容。

## 适用范围

- 适用于本知识库新增通用工程笔记。
- 适用于官方文档、开源项目 README、公开 Issue 的归纳总结。
- 不适用于复制第三方完整源码、保存私有资料或业务项目代码。

## 判断方法

- 资料来源优先看 `sources.md`。
- 新增内容必须能归类为官方文档、开源项目、开源社区资料、实战总结或我的项目经验。
- 每篇文档必须包含排查可用结构，而不是教材式长文。

## 操作步骤

可直接复制以下提示词：

```text
请只在 /Users/dora/Documents/ai-engineering-knowledge-base 内操作。
请学习 sources.md 中列出的公开资料来源，只总结通用工程知识。
不要读取或修改任何业务项目。
不要保存服务器密码、Token、密钥、客户资料、真实数据库数据。
不要写绕过登录、绕过支付、绕过授权、绕过风控、伪造订单、攻击第三方系统的内容。

请为【专题名称】新增或增强知识库文档。
每篇文档必须包含：
问题场景、适用范围、判断方法、操作步骤、验证方法、注意事项、相关命令、参考来源。
命令必须解释用途，危险命令必须标记风险。
完成后更新 99-index/技术索引.md，必要时更新常用命令索引和问题排查索引。
最后执行 find . -name "*.md" -size 0 和 git status。
```

用途：约束 Codex 只学习公开资料并写成可排查的工程笔记。

## 验证方法

- 新文档包含来源类型和参考来源。
- 没有大段复制第三方原文。
- 没有真实密钥、客户资料、生产数据。
- 索引已同步更新。

## 注意事项

- 不确定资料是否合法时，只写“待确认”，不写具体操作。
- 官方文档链接优先于二手博客。
- 不把学习任务扩展到业务项目目录。

## 相关命令

`pwd`、`find . -type f | sort`、`find . -name "*.md" -size 0`、`git status`。
