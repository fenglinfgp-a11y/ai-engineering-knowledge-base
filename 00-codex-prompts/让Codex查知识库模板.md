# 让 Codex 查知识库模板

来源类型：实战总结 / 我的项目经验

## 问题场景

只想让 Codex 在知识库中查找已有资料，给出路径和摘要，不希望它修改文件。

## 适用范围

- 适用于排查前快速找文档。
- 适用于确认某类问题是否已有案例、命令或模板。
- 不适用于让 Codex 读取业务项目文件。

## 判断方法

- 只读需求应明确写“不要修改文件”。
- 查询范围限定在知识库目录。
- 输出应包含文档路径、匹配原因和下一步建议。

## 操作步骤

可直接复制以下提示词：

```text
请只在 /Users/dora/Documents/ai-engineering-knowledge-base 内查找。
不要修改任何文件。
不要读取业务项目目录。
我要排查的问题是：【描述问题】。

请帮我查：
1. 最相关的知识库文档路径。
2. 每篇文档为什么相关。
3. 可先执行的只读排查命令。
4. 哪些操作属于高风险，需要我确认后再做。
```

用途：让 Codex 做知识库检索和排查入口推荐。

## 验证方法

- 输出只包含知识库路径。
- 没有新增、修改、删除文件。
- 推荐命令优先为只读命令。

## 注意事项

- 需要改文件时应另开“最小修改”提示。
- 涉及敏感日志时先脱敏再贴给 Codex。

## 相关命令

`rg "<keyword>" .`、`find . -type f | sort`、`git status --short`。
