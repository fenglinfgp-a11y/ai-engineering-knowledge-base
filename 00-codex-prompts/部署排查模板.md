# 部署排查模板

来源类型：实战总结

## 问题场景

服务器部署失败、服务无法访问、接口 500、Nginx 代理异常、Docker 容器启动失败。

## 判断方法

- 先确认服务是否运行。
- 再确认端口是否监听。
- 再确认 Nginx 或网关是否转发。
- 最后看应用日志和系统日志。

## 操作步骤

可复制模板：

```text
请只做部署排查，不要直接修改生产配置。

环境：
- 系统：<Ubuntu/CentOS>
- 服务类型：<PHP/Java/Node/Docker>
- 域名或端口：<example.com:443>

请按顺序检查：
1. 进程是否运行。
2. 端口是否监听。
3. Nginx 配置是否可解析。
4. 应用日志最近错误。
5. 给出最小修复建议。

所有可能中断服务的命令必须先说明风险，不要直接执行。
```

常用只读命令：

```bash
systemctl status <service>
```

用途：查看系统服务状态，不修改服务。

```bash
ss -lntp
```

用途：查看 TCP 监听端口和对应进程，不修改服务。

```bash
nginx -t
```

用途：检查 Nginx 配置语法，不重载配置。

## 验证方法

- 本机能 `curl http://127.0.0.1:<port>`。
- 服务器能返回预期状态码。
- 日志不再出现同类错误。

## 注意事项

- `systemctl restart <service>` 会中断服务，生产环境执行前必须确认。
- `nginx -s reload` 会重载 Nginx 配置，语法检查通过后再执行。
