# 问题沉淀成案例模板

来源类型：实战总结

## 问题场景

解决一个真实问题后，需要把经验脱敏沉淀到 `cases/`，方便以后复用。

## 适用范围

适用于服务器、后端、数据库、Android、交付等合法开发和排查经验归档。

## 判断方法

- 问题已完成复现、排查、修复和验证。
- 日志、路径、账号、域名、IP、数据库名已脱敏。
- 内容不涉及绕过登录、支付、授权、风控、攻击第三方系统。

## 操作步骤

可复制给 Codex：

```text
请只在 /Users/dora/Documents/ai-engineering-knowledge-base 中操作。

我要把以下问题沉淀成一个脱敏案例：
<粘贴已脱敏的问题描述、环境、日志、排查过程>

要求：
1. 不保存真实 IP、域名、账号、密码、Token、客户资料、真实数据库数据。
2. 案例放到 cases/ 下最合适的子目录。
3. 每条命令都要说明用途，危险命令标注风险。
4. 案例必须包含：来源类型、问题现象、环境背景、初步判断、排查命令、关键日志、解决步骤、验证方式、可复用经验、禁止事项、后续可补充内容。
5. 更新 99-index/技术索引.md 和 99-index/问题排查索引.md。
6. 执行 find . -name "*.md" -size 0 和 git status，但不要 push。
```

## 验证方法

- 新案例能从索引找到。
- 案例只包含占位符和脱敏样例。
- 空 Markdown 检查无输出。

## 注意事项

- 不要把原始日志全文直接写入案例。
- 数据库示例只能使用 `<database_name>`、`<table_name>`、`<id>` 等占位符。

## 相关命令

```bash
find . -name "*.md" -size 0
```

用途：检查是否生成空 Markdown 文件。

```bash
git diff --stat
```

用途：查看本次沉淀影响范围。
