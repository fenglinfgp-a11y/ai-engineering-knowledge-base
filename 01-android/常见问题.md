# Android 常见问题

来源类型：官方文档 / 实战总结

## 问题场景

常见问题包括安装失败、签名不一致、接口无法访问、权限未授权、打包后行为与调试环境不同。

## 判断方法

- 安装失败先看 `INSTALL_FAILED_*`。
- 网络失败先区分 DNS、证书、明文 HTTP、服务端状态码。
- 权限问题先检查 manifest 和运行时授权。

## 操作步骤

```bash
adb install -r app-debug.apk
```

用途：覆盖安装 APK，保留应用数据。风险：签名不同会安装失败，不能用于跨签名覆盖。

```bash
adb install -r -d app-debug.apk
```

用途：允许降级安装测试包。风险：降级可能造成本地数据结构不兼容。

```bash
adb shell pm clear <your.package.name>
```

用途：清空 App 本地数据，排除缓存影响。风险：会删除本地登录态、数据库、缓存文件。

```bash
adb shell am force-stop <your.package.name>
```

用途：强制停止 App，便于重新触发冷启动流程。风险：会中断当前运行状态。

## 验证方法

- 安装命令返回 `Success`。
- 冷启动无崩溃。
- 权限弹窗或系统设置中状态符合预期。
- 接口日志和服务端日志时间能对应。

## 注意事项

- 排查前记录设备型号、Android 版本、ABI、渠道包、签名类型。
- 不要把生产用户日志直接保存到知识库，必须脱敏。

## 适用范围

Android 授权测试、构建、日志分析和合法调试场景。

## 相关命令

`adb devices`、`adb logcat`、`./gradlew tasks`。
