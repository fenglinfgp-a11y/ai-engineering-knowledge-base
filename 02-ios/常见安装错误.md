# iOS 常见安装错误

来源类型：开源社区资料 / 实战总结

## 问题场景

安装 deb 或插件时报错：架构不支持、依赖缺失、包损坏、路径冲突、权限不足。

## 判断方法

- 先看安装器完整错误信息。
- 再看 deb 控制字段。
- 最后检查依赖和目标环境类型。

## 操作步骤

```bash
dpkg-deb -I package.deb
```

用途：确认 deb 元数据能被读取，排除包文件损坏。

```bash
dpkg-deb -c package.deb | grep -i "plist\\|dylib"
```

用途：确认包内是否包含预期的 plist 和 dylib。

```bash
apt-cache policy <dependency-name>
```

用途：查看依赖包是否可用以及候选版本。

## 验证方法

- 安装器不再报 `architecture`、`dependency`、`bad deb` 类错误。
- 安装后文件出现在预期路径。
- 重启相关进程后功能在授权测试范围内可用。

## 注意事项

- 不要为了安装成功而删除依赖约束；依赖约束是运行稳定性的保护。
- 不要记录规避签名、授权或安全检测的操作。

## 适用范围

iOS deb、dylib、rootless/rootful 兼容性和合法安装排查。

## 相关命令

`dpkg-deb -I`、`dpkg-deb -c`、`file`、`otool -L`。
