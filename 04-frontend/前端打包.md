# 前端打包

来源类型：官方文档 / 实战总结

## 问题场景

前端打包失败、部署后白屏、静态资源 404、接口地址错误。

## 判断方法

- 先看 `package.json` 的 build 脚本。
- 确认 Node 版本和包管理器。
- 查看构建输出目录。
- 浏览器 Console 和 Network 是否有资源加载错误。

## 操作步骤

```bash
node -v
```

用途：查看 Node.js 版本。

```bash
npm ci
```

用途：按 lock 文件干净安装依赖，适合 CI。风险：会删除现有 `node_modules/` 后重装。

```bash
npm run build
```

用途：执行项目定义的生产构建。

```bash
npx serve dist
```

用途：本地预览静态构建产物，确认部署前可访问。

## 验证方法

- 构建命令成功。
- 本地预览页面无白屏。
- Network 面板没有关键 JS/CSS 404。
- 接口请求指向正确环境。

## 注意事项

- `dist/` 是构建产物，不等于源码交付。
- 环境变量命名受框架限制，例如 Vite 常用 `VITE_` 前缀。

## 适用范围

Vue/Nuxt 前端项目结构、构建、部署和后台页面维护。

## 相关命令

`node -v`、`npm install`、`npm run dev`、`npm run build`。
