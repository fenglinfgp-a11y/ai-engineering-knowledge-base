# 后台页面维护

来源类型：实战总结

## 问题场景

后台管理页面菜单不显示、权限异常、表格接口失败、构建后页面空白。

## 判断方法

- 菜单是否来自前端路由、后端接口或权限配置。
- 用户角色是否拥有页面权限。
- 接口返回字段是否符合表格组件预期。
- 浏览器 Console 是否报错。

## 操作步骤

```bash
npm run dev
```

用途：启动本地后台页面，便于打开开发者工具排查。

```bash
grep -R "菜单名称或路由关键字" src
```

用途：在源码中查找菜单、路由或页面组件位置。

```bash
npm run build
```

用途：验证修改后是否还能生产构建。

## 验证方法

- 使用测试账号登录后能看到目标菜单。
- 页面接口返回 200，表格能渲染数据。
- 刷新页面后路由不 404。
- 生产构建无错误。

## 注意事项

- 权限问题要区分前端隐藏菜单和后端接口鉴权。
- 不要在前端硬编码真实管理员账号或 Token。

## 适用范围

Vue/Nuxt 前端项目结构、构建、部署和后台页面维护。

## 相关命令

`node -v`、`npm install`、`npm run dev`、`npm run build`。
