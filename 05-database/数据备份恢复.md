# 数据备份恢复

来源类型：官方文档 / 实战总结  
参考来源：MySQL mysqldump 官方文档

## 问题场景

需要备份数据库、迁移测试环境、恢复误删数据或验证交付 SQL。

## 判断方法

- 先确认备份对象是结构、数据，还是二者都要。
- 恢复前确认目标库是否为空或可覆盖。
- 生产恢复必须先在测试库演练。

## 操作步骤

```bash
mysqldump -h <host> -u <user> -p --databases <database_name> > backup.sql
```

用途：备份指定数据库结构和数据，密码交互输入。

```bash
mysqldump -h <host> -u <user> -p --no-data <database_name> > schema.sql
```

用途：只导出表结构，不导出数据，适合源码交付。

```bash
mysql -h <host> -u <user> -p <database_name> < backup.sql
```

用途：把 SQL 导入目标库。风险：可能覆盖或插入大量数据，执行前确认目标库和备份来源。

## 验证方法

- 导入后 `SHOW TABLES;` 能看到预期表。
- 核心表行数与备份前记录一致。
- 应用连接测试库能正常启动。

## 注意事项

- 生产备份文件可能包含敏感数据，不能直接提交到 Git。
- 交付包优先提供结构 SQL 或脱敏样例数据。

## 适用范围

MySQL 表结构、备份恢复、SQL 排查和脱敏数据维护。

## 相关命令

`mysql`、`mysqldump`、`SHOW CREATE TABLE`、`EXPLAIN`。
