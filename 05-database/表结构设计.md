# 表结构设计

来源类型：官方文档 / 实战总结  
参考来源：MySQL 官方文档

## 问题场景

新建业务表或接手旧表时，需要判断字段、索引、主键、时间字段是否合理。

## 判断方法

- 是否有稳定主键。
- 查询条件是否有对应索引。
- 金额是否避免使用浮点类型。
- 是否有 `created_at`、`updated_at` 等审计字段。

## 操作步骤

```sql
SHOW CREATE TABLE <table_name>\G
```

用途：查看完整建表 SQL，包括字符集、索引、约束。

```sql
EXPLAIN SELECT * FROM <table_name> WHERE <column_name> = 'test';
```

用途：查看查询是否使用索引。示例值必须使用测试数据。

```sql
SHOW INDEX FROM <table_name>;
```

用途：查看表索引列表和索引字段。

## 验证方法

- 高频查询能命中合适索引。
- 字段类型能覆盖业务范围且不过度浪费。
- 唯一约束能阻止重复关键数据。

## 注意事项

- 生产大表加索引可能锁表或影响性能，执行前必须评估。
- 不要把真实用户字段样例写进文档。

## 适用范围

MySQL 表结构、备份恢复、SQL 排查和脱敏数据维护。
