# 宝塔面板

来源类型：官方文档 / 实战总结  
参考来源：aaPanel / 宝塔公开文档

## 问题场景

使用面板管理 Nginx、PHP、MySQL、SSL 时，网站无法访问或配置不一致。

## 判断方法

- 面板中站点配置与服务器实际配置是否一致。
- Nginx/PHP/MySQL 服务是否运行。
- 网站目录、运行用户、PHP 版本是否正确。

## 操作步骤

```bash
bt status
```

用途：查看宝塔面板服务状态，具体命令以服务器安装版本为准。

```bash
sudo systemctl status nginx
```

用途：查看 Nginx 是否正常运行。

```bash
sudo systemctl status mysql
```

用途：查看 MySQL 服务状态，部分系统服务名可能是 `mysqld` 或 `mariadb`。

```bash
sudo nginx -t
```

用途：检查面板生成的 Nginx 配置是否可用。

## 验证方法

- 面板服务能打开。
- 网站绑定目录正确。
- Nginx 配置检查通过。
- 站点访问返回预期页面。

## 注意事项

- 面板里保存的数据库密码、FTP 密码、面板入口地址不能写入知识库。
- 面板操作和手写配置混用时，要记录最终生效位置。

## 适用范围

Ubuntu、Nginx、Docker、Java 服务和防火墙等服务器部署排查。

## 相关命令

`ss -lntp`、`nginx -t`、`systemctl status`、`docker ps`、`ufw status`。
