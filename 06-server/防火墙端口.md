# 防火墙端口

来源类型：官方文档 / 实战总结  
参考来源：Ubuntu UFW 官方文档

## 问题场景

服务本机能访问，外网无法访问，可能是防火墙、安全组或监听地址问题。

## 判断方法

- 服务是否监听 `0.0.0.0` 或指定公网网卡。
- 服务器防火墙是否开放端口。
- 云厂商安全组是否开放端口。

## 操作步骤

```bash
ss -lntp
```

用途：查看 TCP 监听端口和进程。

```bash
sudo ufw status numbered
```

用途：查看 UFW 防火墙规则及编号。

```bash
sudo ufw allow 8080/tcp
```

用途：开放 8080 TCP 端口。风险：会扩大网络暴露面，执行前确认服务需要公网访问。

```bash
sudo ufw delete <rule_number>
```

用途：按编号删除防火墙规则。风险：可能关闭业务端口，执行前确认编号对应规则。

## 验证方法

- 本机 `curl http://127.0.0.1:<port>` 成功。
- 服务器公网地址加端口能从外部访问。
- `ufw status` 显示目标端口开放。

## 注意事项

- 只开放必要端口。
- 数据库、Redis、管理面板不应随意暴露到公网。

## 适用范围

Ubuntu、Nginx、Docker、Java 服务和防火墙等服务器部署排查。

## 相关命令

`ss -lntp`、`nginx -t`、`systemctl status`、`docker ps`、`ufw status`。
