# 交付包清理规则

来源类型：实战总结 / 我的项目经验

## 问题场景

项目交付前，需要清理缓存、日志、密钥、依赖目录和真实数据。

## 判断方法

- 是否包含 `.env`、私钥、证书、数据库备份。
- 是否包含 `node_modules`、`vendor`、`target`、`dist`。
- 是否包含日志、缓存、上传文件、临时文件。

## 操作步骤

```bash
find . -type f \\( -name ".env" -o -name "*.key" -o -name "*.pem" -o -name "*.p12" \\)
```

用途：查找可能包含密钥或环境变量的文件。

```bash
find . -type d \\( -name node_modules -o -name vendor -o -name target -o -name dist \\) -prune -print
```

用途：查找常见依赖目录和构建产物目录。

```bash
find . -type f \\( -name "*.log" -o -name "*.sql" -o -name "*.zip" \\) | head -100
```

用途：查找日志、SQL、压缩包等需要人工确认的文件。

## 验证方法

- 交付包不包含真实密钥和真实数据。
- 依赖可通过包管理器重新安装。
- 构建产物可通过源码重新生成。
- README 中说明如何配置环境变量示例。

## 注意事项

- `rm -rf <path>` 是高风险删除命令，可能永久删除文件，执行前必须确认路径。
- 清理前先备份原始交付包。

## 适用范围

源码交付、交付包清理、可开发性判断和客户验收准备。
