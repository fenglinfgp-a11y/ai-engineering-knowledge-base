# 前后端数据库三件套交付流程

来源类型：实战总结 / 我的项目经验  
参考来源：Spring Boot、Vue、Nuxt、MySQL 官方文档

## 问题场景

交付一个 Web 项目时，需要把前端、后端、数据库结构和部署说明整理成可验收的最小包。

## 适用范围

- 适用于前端源码、后端源码、数据库结构 SQL 的组合交付。
- 适用于脱敏测试数据和部署说明。
- 不交付生产密钥、客户资料和真实数据库数据。

## 判断方法

- 前端是否能安装依赖并构建。
- 后端是否能安装依赖并启动。
- 数据库结构是否能导入空测试库。
- 前后端 API 地址配置是否使用示例占位符。
- README 是否写明启动顺序和验证接口。

## 操作步骤

```bash
find frontend backend database -maxdepth 3 -type f | sort
```

用途：查看三件套目录结构，目录名可按实际交付包调整。

```bash
mysqldump -h <host> -u <user> -p --no-data <database_name> > database/schema.sql
```

用途：导出数据库结构 SQL。风险：确认只导出结构，不能把生产数据写入交付包。

```bash
cd backend && ./mvnw test
```

用途：在后端测试环境运行测试，具体命令按项目技术栈调整。

```bash
cd frontend && npm run build
```

用途：验证前端能构建。

```bash
find . -type f \\( -name ".env" -o -name "*.key" -o -name "*.pem" -o -name "*.log" \\)
```

用途：交付前扫描敏感文件。

## 验证方法

- 前端构建通过。
- 后端测试或启动通过。
- 数据库结构导入空测试库通过。
- README 能让接手方完成本地启动。
- 敏感文件扫描结果为空或已解释并脱敏。

## 注意事项

- 交付顺序建议为数据库结构、后端服务、前端配置和构建。
- API 地址、数据库连接、第三方密钥全部使用占位符。
- 如果缺少任一件，需要在交付说明中明确风险和补齐项。

## 相关命令

`find`、`mysqldump --no-data`、`./mvnw test`、`npm run build`、`grep -RniE`。
