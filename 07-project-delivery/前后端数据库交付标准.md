# 前后端数据库交付标准

来源类型：实战总结 / 我的项目经验

## 问题场景

判断一个项目交付包是否能让接手方继续开发、部署和维护。

## 判断方法

- backend：后端源码、依赖说明、启动命令、环境变量示例。
- frontend：前端源码、包管理器、构建命令、部署目录。
- database：结构 SQL、迁移文件、初始化说明。

## 操作步骤

```bash
find . -maxdepth 2 -type f | sort
```

用途：快速查看交付包顶层文件结构。

```bash
git status --short
```

用途：确认是否有未提交或临时文件混入。

```bash
du -sh *
```

用途：查看各目录大小，发现异常大的日志、缓存、依赖目录。

## 验证方法

- 后端能安装依赖并启动。
- 前端能安装依赖并构建。
- 数据库能在空库导入结构。
- README 写清楚启动、构建、部署步骤。

## 注意事项

- 不交付真实 `.env`、生产数据库备份、密钥、客户资料。
- `node_modules`、`vendor`、`target`、`dist` 通常不作为源码核心交付。

## 适用范围

源码交付、交付包清理、可开发性判断和客户验收准备。

## 相关命令

`find`、`du -sh`、`git status --short`、敏感字段扫描命令。
