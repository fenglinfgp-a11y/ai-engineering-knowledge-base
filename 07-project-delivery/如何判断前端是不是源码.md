# 如何判断前端是不是源码

来源类型：官方文档 / 实战总结 / 我的项目经验  
参考来源：Vue 官方文档、Nuxt 官方文档、npm 文档

## 问题场景

收到前端交付包后，需要判断它是 Vue/Nuxt/React 等可开发源码，还是只有 `dist/`、静态 HTML、构建产物。

## 适用范围

- 适用于前端源码验收、交付包检查和部署前判断。
- 适用于公开项目结构知识和脱敏交付经验。
- 不复制业务页面源码或客户素材。

## 判断方法

- 可开发源码通常包含 `package.json`、`src/`、`vite.config.*`、`nuxt.config.*`。
- Vue 项目常见 `src/components`、`src/router`、`src/views`。
- Nuxt 项目常见 `pages/`、`components/`、`app.vue`、`nuxt.config.ts`。
- 只有 `dist/`、`.output/`、静态 JS/CSS/HTML 时通常是构建产物。
- 可开发源码应能安装依赖并执行构建命令。

## 操作步骤

```bash
find . -maxdepth 3 -type f | sort
```

用途：查看交付包文件结构。

```bash
find . -maxdepth 3 -type f \\( -name "package.json" -o -name "vite.config.*" -o -name "nuxt.config.*" \\)
```

用途：查找前端依赖文件和构建配置。

```bash
find . -maxdepth 3 -type d \\( -name "src" -o -name "pages" -o -name "components" -o -name "dist" -o -name ".output" \\)
```

用途：判断源码目录和构建产物目录是否存在。

```bash
npm run build
```

用途：在测试环境验证是否能构建。风险：会生成构建产物并可能访问配置中的测试接口，执行前检查 `.env`。

## 验证方法

- 能找到 `package.json` 和源码目录。
- `package.json` 中有明确的 `scripts`。
- 测试环境安装依赖和构建通过。
- 交付说明包含 Node 版本、包管理器、构建命令和部署目录。

## 注意事项

- `node_modules` 不是源码核心，不建议作为交付判断依据。
- `.env.production` 可能包含敏感地址或密钥，交付前必须脱敏。
- 只记录结构判断，不搬运业务页面代码。

## 相关命令

`find`、`npm run build`、`du -sh`、`git status --short`。
