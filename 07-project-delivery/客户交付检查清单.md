# 客户交付检查清单

来源类型：实战总结 / 我的项目经验

## 问题场景

项目准备交付给客户或后续维护方，需要确认资料完整、风险清楚、可部署可运行。

## 判断方法

- 是否包含源码、数据库结构、部署说明。
- 是否有测试账号和测试数据说明，且不包含真实敏感数据。
- 是否说明运行环境和版本。
- 是否有已知问题和未交付范围。

## 操作步骤

```bash
find . -maxdepth 2 -type f | sort
```

用途：输出交付包文件清单，便于人工核对。

```bash
grep -R "password\\|token\\|secret\\|AKIA\\|PRIVATE KEY" . --exclude-dir=.git --exclude-dir=node_modules
```

用途：粗略扫描可能泄露的敏感字段。风险：可能输出敏感内容，结果不要外发。

```bash
git log --oneline -5
```

用途：查看最近提交记录，确认交付版本来源。

## 验证方法

- 按 README 能在测试环境部署。
- backend、frontend、database 三件套齐全。
- 交付包扫描未发现真实密钥。
- 客户可根据说明独立启动或验收。

## 注意事项

- 测试账号密码也应单独安全传递，不写进公开仓库。
- 已知限制要明示，不要让接手方靠猜。

## 适用范围

源码交付、交付包清理、可开发性判断和客户验收准备。

## 相关命令

`find`、`du -sh`、`git status --short`、敏感字段扫描命令。
