# 源码交付敏感信息清理

来源类型：官方文档 / 实战总结 / 我的项目经验  
参考来源：GitHub Docs、Docker 官方文档

## 问题场景

源码交付前，需要清理 `.env`、私钥、Token、真实数据库备份、日志、客户资料等敏感内容。

## 适用范围

- 适用于交付包打包前的本地扫描和人工复核。
- 适用于公开安全原则和脱敏流程。
- 不保存真实密钥、Token、客户资料和生产数据。

## 判断方法

- 查找 `.env`、证书私钥、数据库备份、日志文件。
- 查找常见敏感字段名，例如 password、secret、token、access_key。
- 检查 README 是否改用占位符。
- 检查 `.gitignore` 是否排除本地配置和运行产物。
- 若敏感信息已经进入 Git 历史，停止交付并按平台安全流程处理，不在知识库记录具体密钥。

## 操作步骤

```bash
find . -type f \\( -name ".env" -o -name "*.pem" -o -name "*.key" -o -name "*.p12" -o -name "*.log" -o -name "*.sql" \\)
```

用途：查找常见敏感文件和高风险文件。

```bash
grep -RniE "password|passwd|secret|token|access[_-]?key|private[_-]?key" . --include="*.md" --include="*.yml" --include="*.yaml" --include="*.json" --include="*.env*"
```

用途：扫描文本配置中的敏感关键词，命中后逐条人工判断。

```bash
git status --short
```

用途：确认清理后的工作区状态。

```bash
git diff --stat
```

用途：查看本次清理影响范围，避免误删源码。

## 验证方法

- 交付包不包含真实 `.env`、私钥、Token、生产 SQL、日志和客户资料。
- 示例配置使用 `<HOST>`、`<USER>`、`<PASSWORD>`、`<TOKEN>` 等占位符。
- `.gitignore` 已覆盖本地配置、依赖目录、构建产物和日志。
- 接手方能根据 `.env.example` 自行配置测试环境。

## 注意事项

- 不要把扫描出来的真实密钥复制到知识库或聊天记录。
- 删除文件前确认是否为唯一源码文件，避免误删。
- 已泄露的密钥应轮换，不能只从交付包删除。

## 相关命令

`find`、`grep -RniE`、`git status --short`、`git diff --stat`。
