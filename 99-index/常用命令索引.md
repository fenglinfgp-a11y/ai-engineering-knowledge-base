# 常用命令索引

来源类型：实战总结

## 问题场景

需要快速查找常用排查命令及其用途。

## 判断方法

- Android 用 `adb`、`gradlew`。
- 后端服务用 `curl`、`systemctl`、`journalctl`。
- 前端用 `npm`、`node`。
- 数据库用 `mysql`、`mysqldump`。
- 服务器用 `ss`、`nginx`、`ufw`、`docker`。
- 分主题命令库优先看 `commands/` 目录。

## 操作步骤

```bash
adb devices
```

用途：确认 Android 测试设备连接状态。

```bash
adb logcat -v time
```

用途：查看 Android 设备实时日志。

```bash
./gradlew assembleDebug --stacktrace
```

用途：构建 Android Debug 包并输出详细异常。

```bash
curl -i http://127.0.0.1:<port>/health
```

用途：查看本机服务接口响应状态码和响应头。

```bash
systemctl status <service-name>
```

用途：查看系统服务状态。

```bash
journalctl -u <service-name> -n 200 --no-pager
```

用途：查看 systemd 服务最近 200 行日志。

```bash
nginx -t
```

用途：检查 Nginx 配置语法。

```bash
ss -lntp
```

用途：查看 TCP 监听端口和对应进程。

```bash
docker ps -a
```

用途：查看所有 Docker 容器状态。

```bash
mysqldump -h <host> -u <user> -p --no-data <database_name> > schema.sql
```

用途：只导出 MySQL 表结构，适合源码交付。

## 验证方法

- 命令输出能帮助判断当前状态。
- 写操作前已经确认目标环境、路径、库名或服务名。

## 注意事项

- `restart`、`down`、`DELETE`、`DROP`、`TRUNCATE`、`rm -rf` 都属于高风险动作。
- 命令中的 `<host>`、`<user>`、`<port>` 必须替换成测试或脱敏值。
