# 前后端数据库三件套检查案例

来源类型：实战总结 / 我的项目经验

## 问题现象

项目交付包无法判断是否具备 backend、frontend、database 三件套，接手后无法启动。

## 环境背景

- 后端：`<backend-stack>`
- 前端：`<frontend-stack>`
- 数据库：`<database-type>`
- 交付目录：`<delivery-package>`

## 初步判断

分别确认后端源码、前端源码、数据库结构、环境变量示例、部署说明是否齐全。

## 排查命令

```bash
find . -maxdepth 3 -type f | sort
```

用途：查看交付包文件结构。

```bash
find . -maxdepth 3 -name "package.json" -o -name "pom.xml" -o -name "composer.json"
```

用途：查找前后端依赖声明文件。

```bash
find . -type f \( -name "*.sql" -o -name "*migration*" \) | head
```

用途：查找数据库结构 SQL 或迁移文件。

## 关键日志

```text
缺少 database/schema.sql，无法初始化空数据库
```

含义：数据库结构不可复现，交付不完整。

## 解决步骤

1. 检查 backend 是否有源码和启动说明。
2. 检查 frontend 是否有源码和构建说明。
3. 检查 database 是否有结构 SQL 或迁移文件。
4. 检查 `.env.example` 是否完整。
5. 在测试环境按说明走一遍启动流程。

## 验证方式

- 后端能启动。
- 前端能构建。
- 空库能初始化结构。
- 三件套说明能让接手方复现环境。

## 可复用经验

可交付源码的最低标准是“能重新搭建”，不是“某台机器上能跑”。

## 禁止事项

- 不交付真实数据库数据。
- 不交付真实密钥。
- 不用构建产物替代源码。
