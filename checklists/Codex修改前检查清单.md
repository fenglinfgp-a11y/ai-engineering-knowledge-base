# Codex 修改前检查清单

来源类型：实战总结 / 我的项目经验

## 问题场景

让 Codex 修改文件前，需要确认工作目录、修改范围和禁止事项，避免误改业务项目。

## 适用范围

- 本知识库维护。
- 其他项目中需要 Codex 只读分析或最小改动时的前置检查。

## 判断方法

- 是否明确当前目录。
- 是否明确允许修改的文件范围。
- 是否明确禁止保存敏感信息。
- 是否明确验证命令。

## 操作步骤

- [ ] 先确认 `pwd`。
- [ ] 先查看 `git status --short`。
- [ ] 明确“只分析”还是“允许修改”。
- [ ] 明确禁止修改的目录和文件。
- [ ] 明确验证命令。
- [ ] 修改前说明将要编辑哪些文件。
- [ ] 修改后输出变更摘要和验证结果。

## 相关命令

```bash
pwd
```

用途：确认当前工作目录。

```bash
git status --short
```

用途：查看当前仓库是否已有改动。

```bash
git diff --stat
```

用途：查看修改文件和行数概览。

## 验证方法

- Codex 只在允许目录内修改。
- 没有无关文件变更。
- 输出包含验证命令和结果。

## 注意事项

- 不要让 Codex 保存服务器密码、Token、客户资料、真实数据库数据。
- 对生产环境问题，先只读排查，再确认是否执行变更。
