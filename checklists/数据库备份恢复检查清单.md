# 数据库备份恢复检查清单

来源类型：官方文档 / 实战总结

## 问题场景

执行数据库备份、迁移、恢复或清理前，需要避免误库、误表、误删。

## 适用范围

- MySQL 测试库和授权生产库维护。
- 结构备份、数据备份、恢复演练、日志表清理。

## 判断方法

- 是否确认目标库。
- 是否有备份文件。
- 是否在测试环境验证过恢复。
- 是否知道操作会影响哪些表和行。

## 操作步骤

- [ ] 确认当前连接环境：测试 / 生产。
- [ ] 确认库名、表名、时间范围。
- [ ] 先导出结构或完整备份。
- [ ] 恢复前确认目标库为空或可覆盖。
- [ ] 删除前先 `SELECT COUNT(*)`。
- [ ] 大表操作安排低峰期。
- [ ] 记录执行命令和验证结果，禁止记录真实密码。

## 相关命令

```bash
mysqldump -h <host> -u <user> -p --databases <database_name> > backup.sql
```

用途：备份指定数据库结构和数据。

```bash
mysqldump -h <host> -u <user> -p --no-data <database_name> > schema.sql
```

用途：只备份数据库结构。

```sql
SELECT COUNT(*) FROM <table_name> WHERE created_at < '2026-01-01';
```

用途：删除前确认影响行数。

```bash
mysql -h <host> -u <user> -p <database_name> < backup.sql
```

用途：恢复 SQL 到目标库。风险：可能覆盖或插入大量数据，必须确认目标库。

## 验证方法

- 恢复后表数量、核心表行数符合预期。
- 应用连接测试库正常。
- 删除范围符合保留策略。

## 注意事项

- 备份文件可能包含敏感数据，不能提交 Git。
- 不允许执行无条件删除。
