# 服务器部署前检查清单

来源类型：实战总结 / 我的项目经验

## 问题场景

上线或迁移服务前，需要避免端口、依赖、配置、备份、回滚方案遗漏。

## 适用范围

- Ubuntu / Nginx / Docker / Java / PHP / Node 服务部署前检查。
- 只用于合法授权服务器和测试环境。

## 判断方法

- 环境是否准备好。
- 服务是否可启动。
- 端口和防火墙是否开放。
- 备份和回滚是否准备好。

## 操作步骤

- [ ] 确认服务器系统版本。
- [ ] 确认部署用户和目录权限。
- [ ] 确认运行时版本：JDK、PHP、Node、MySQL、Docker。
- [ ] 确认 `.env.example` 已准备，真实 `.env` 不提交到仓库。
- [ ] 确认数据库已备份。
- [ ] 确认 Nginx 配置已通过语法检查。
- [ ] 确认防火墙和云安全组开放必要端口。
- [ ] 确认回滚包或上一个版本可恢复。

## 相关命令

```bash
lsb_release -a
```

用途：查看 Ubuntu 系统版本。

```bash
ss -lntp
```

用途：查看端口监听情况。

```bash
sudo nginx -t
```

用途：检查 Nginx 配置语法。

```bash
sudo ufw status numbered
```

用途：查看防火墙规则和编号。

## 验证方法

- 本机健康检查接口返回预期状态。
- 外部访问域名或端口正常。
- 应用日志无启动异常。
- 回滚路径已确认。

## 注意事项

- 重启服务、reload Nginx、开放端口都可能影响线上访问。
- 不要把真实服务器密码、Token、私钥写入清单。
