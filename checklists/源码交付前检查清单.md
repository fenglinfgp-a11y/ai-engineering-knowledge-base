# 源码交付前检查清单

来源类型：实战总结 / 我的项目经验

## 问题场景

交付源码前，需要确认代码可开发、可部署、无敏感信息。

## 适用范围

- 前后端数据库三件套交付。
- 客户验收、二次开发、内部移交。

## 判断方法

- 是否包含源码而不是只有构建产物。
- 是否包含数据库结构和部署说明。
- 是否排除了真实密钥、日志、缓存和客户数据。

## 操作步骤

- [ ] backend：源码、依赖文件、启动说明完整。
- [ ] frontend：源码、依赖文件、构建命令完整。
- [ ] database：结构 SQL 或迁移文件完整。
- [ ] README：写明环境版本、启动、构建、部署步骤。
- [ ] `.env.example`：只放占位符。
- [ ] 已清理日志、缓存、真实上传文件。
- [ ] 已检查是否有密钥、Token、私钥。
- [ ] 已验证测试环境能启动或构建。

## 相关命令

```bash
find . -maxdepth 3 -type f | sort
```

用途：查看交付包文件结构。

```bash
find . -type f \( -name ".env" -o -name "*.pem" -o -name "*.key" \)
```

用途：查找可能包含敏感信息的文件。

```bash
du -sh *
```

用途：查看目录大小，发现异常大的依赖或日志目录。

## 验证方法

- 后端能启动。
- 前端能打包。
- 空库能导入数据库结构。
- 敏感信息扫描没有真实密钥。

## 注意事项

- 交付测试账号应通过安全渠道单独传递。
- 不要交付真实生产数据库。
