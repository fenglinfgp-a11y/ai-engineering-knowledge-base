# 禁止写入知识库的内容

来源类型：实战总结 / 我的项目经验

## 问题场景

知识库长期维护时，需要明确哪些内容永远不能写入，避免泄露或越界。

## 适用范围

适用于本仓库所有 Markdown、Issue、PR、案例、命令和流程文档。

## 判断方法

如果内容会暴露真实系统、真实账号、真实客户、真实数据、私有代码或绕过类做法，就不能写入。

## 操作步骤

禁止写入：

- 服务器真实 IP、密码、Token、密钥。
- 客户资料、客户名称、客户业务数据。
- 真实数据库数据、完整生产备份、真实 SQL 查询结果。
- 授权私钥、证书私钥、License 私钥。
- 第三方账号、Cookie、Session、OAuth 凭据。
- 闭源项目完整代码或私有仓库内容。
- 绕过登录、绕过支付、绕过授权、绕过风控、伪造订单、攻击第三方系统的做法。
- 未授权 App、服务、系统的敏感分析内容。

## 相关命令

```bash
git status --short
```

用途：提交前确认是否有不该提交的文件。

```bash
find . -type f \( -name ".env" -o -name "*.pem" -o -name "*.key" -o -name "*.sql" \)
```

用途：查找高风险文件。风险：结果可能暴露敏感路径，不要外发。

## 验证方法

- 高风险文件不进入 Git。
- 文档只保留脱敏示例。
- 所有绕过类内容被排除。

## 注意事项

- 合法 Billing 调试只记录官方测试流程和日志分析，不写伪造订单。
- Hook/插件调试只记录授权测试和兼容性排查，不写规避检测。
- 真实客户信息不要以截图形式保存。
