# 脱敏规则

来源类型：实战总结 / 我的项目经验

## 问题场景

把排查经验写入知识库前，需要把真实环境信息替换成安全占位符。

## 适用范围

适用于所有 `cases/`、Issue 模板、交付记录、日志片段、命令示例。

## 判断方法

凡是能定位到真实服务器、真实客户、真实账号、真实数据、真实密钥的信息，都必须替换。

## 操作步骤

| 原始内容类型 | 脱敏写法 |
| --- | --- |
| 服务器 IP | `<server-ip>` |
| 域名 | `<example-domain>` 或 `example.com` |
| 用户名 | `<user>` |
| 密码 | `<PASSWORD>` |
| Token | `<TOKEN>` |
| 私钥 | `<PRIVATE_KEY>` |
| 数据库名 | `<database_name>` |
| 表名 | `<table_name>` |
| 客户名 | `<client-name>` |
| 订单号 | `<order-id>` |
| 手机号/邮箱 | `<phone>` / `<email>` |

## 相关命令

```bash
grep -R "password\\|token\\|secret\\|PRIVATE KEY\\|AKIA" . --include="*.md"
```

用途：粗略扫描可能敏感的关键词。风险：如果文档已含敏感内容，输出也可能敏感，不要外发。

```bash
grep -R "[0-9]\\{1,3\\}\\.[0-9]\\{1,3\\}\\.[0-9]\\{1,3\\}\\.[0-9]\\{1,3\\}" . --include="*.md"
```

用途：粗略扫描 IPv4 地址，结果需要人工判断是否为示例值。

## 验证方法

- 文档里没有真实 IP、真实客户名、真实账号、真实密钥。
- 日志片段只保留排查必要字段。
- 命令示例使用占位符。

## 注意事项

- 脱敏不是简单删除，必须保留足够的排查语义。
- 不确定是否敏感时，按敏感处理。
- 不要把完整生产日志粘贴进知识库。
